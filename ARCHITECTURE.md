# Архитектура — КаниКар

## Общ преглед
Едностраничено приложение (`index.html`) с inline CSS/JS. Каталогът, филтрите
и количката се управляват изцяло client-side — няма видим backend в repo-то.

## Основни UI компоненти
| Компонент | Отговорност |
|---|---|
| `.filters` (`searchInput`, `typeFilter`, `constructFilter`, `availFilter`, `brandFilterChips`) | Търсене и многокритериално филтриране |
| `btnGrid` / `btnList` | Превключване на изгледа на каталога |
| `catalogToggle` / `partsToggle` / `serviceToggle` | Разгъваеми секции |
| `modalOverlay` (full product card) | Детайли, поръчка, запитване за оферта |
| `cartOverlay` + `btnCheckout` | Количка и checkout flow |
| Auth модал (`authBtn`, `openAuthModal`) | Вход/регистрация |
| `ownerLogin()` | Owner/admin достъп |

## Роли
- **Гост** — разглежда каталога, филтрира, добавя в количка.
- **Регистриран потребител** — поръчки, история (ако е имплементирано).
- **Owner** — административни функции.

## Препоръки за бъдещо развитие
- При растеж на каталога — обмисли извеждане на продуктовите данни в отделен
  JSON/API вместо вградени в скрипта.
- Ако checkout flow-ът стане по-сложен (плащания), обмисли backend за
  сигурна обработка — не дръж платежна логика само client-side.
